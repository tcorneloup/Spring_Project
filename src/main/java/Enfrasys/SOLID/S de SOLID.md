-> Principes SOLID
    
 S = Single Responsability, où une classe ne doit avoir qu'une suele responsabilité
 O = Open/Close Principle, Ouvert à l'extension mais fermé à la modification
 L = Liskov Principle, une sous-classe peut remplacer sa classe mère sans bug
 I = Interface segregation pour des interfaces petites et specifiques
 D = Dependency Inversion Principle afin que la dépendence soit sur des interface et non sur des classes concrètes
